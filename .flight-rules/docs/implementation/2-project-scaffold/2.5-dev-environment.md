# 2.5 Dev Environment

## Overview

Set up the development environment configuration, scripts, and documentation that helps users get started quickly. Most of this is r9stack-provided content that builds on what the CLIs create.

## What Comes From CLIs vs r9stack

| Component | Source |
|-----------|--------|
| Base package.json scripts | TanStack Start CLI |
| Convex dev script | Convex CLI |
| Combined dev workflow | r9stack (merges both) |
| Environment templates | r9stack |
| Service setup instructions | r9stack |
| Project README | r9stack |

## 2.5.6 Project Versioning

**Goal:** Include versioning infrastructure in generated projects following the flight-rules pattern.

**Reference:** Follow the pattern from [flight-rules](https://github.com/ryanpacker/flight-rules)

**Steps:**
- Add `project_version: 0.1.0` header to key project files (e.g., README.md or a project manifest)
- Create scripts/sync-version.js in generated project
- Configure npm "version" script to sync version across project files
- Include `r9stack_version: X.Y.Z` to track which r9stack version generated the project
- Document version management in project README
